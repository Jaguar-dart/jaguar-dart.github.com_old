Installing Jaguar

Jaguar uses `jaguar` command for building, watching for changes and creating boilerplates. Execute the following command to install `jaguar` command: 

  pub global activate jaguar

Now `jaguar` command must be included in your system path. Type `jaguar` in your command line to list the list of commands jaguar supports.

More information about jaguar command can be found in jaguar command referece.

Hello world

In this section, we will clone a tiny example project from github and analyse basics of Jaguar. 

  git clone https://github.com/Jaguar-dart/boilerplate.git

Routes, RouteGroups and Api

In Jaguar, related routes are composed into a RouteGroup and related RouteGroups are composed into an Api.

Note:This is a simple project for demonstration. So it doesn't have models, static files, etc.

The RouteGroups and Api reside in `lib/api.dart` dart file. Notice the following line `part 'api.g.dart';`. Later, we will generate `api.g.dart` using the command `jaguar build`. `api.g.dart` will have declaration of `_$JaguarExampleApi` and `_$JaguarExampleRouteGroup`.

The main function

Finally the server is configured to run on specific host, port and serve specified Apis. This can be found in `bin/server.dart` dart file.

Building RouteGroup and Api

jaguar.yaml

`jaguar` command uses `jaguar.yaml` located in root directory of the project for configurations. `jaguar.yaml` is used to convey the files to be built by jaguar and the location of the server main file. `apis` sequence is used to list the list of files jaguar has to build. We will learn what scalar `bin` does in the section below.

Building

Type the following command from the root directory of the project to build the project: 

  jaguar build

`jaguar` command parses the `jaguar.yaml` file and generates all the necessary dart files. In this case `api.g.dart`. Take a peek at the baremetal performant dart code Jaguar generated for you from the annotations. Pure magic!
