Documentation for Jaguar command

build

Build command generates Routehandler for RouteGroupss and Apis.

It gets its configuration from jaguar.yaml file present in root directory of the project. The jaguar.yaml should contain a sequence named apis and a scalar named bin. The apis list the list of files in the project jaguar must build. bin specifies the main file of the project.

Flags
Build command doesn't accept any flags.

Usage
  jaguar build

Example jaguar.yaml
bin:
    'bin/server.dart'

  apis:
    - 'lib/api.dart'

watch

Watch command does what build does. But build executes once and finishes. While watch command watches for changes in code and generates everytime a project content is changed.

It also has the capability to run the server and restart when the project content are changed. It gets which dart file to run from bin scalar in jaguar.yaml file.

Flags
Watch command doesn't accept any flags.

Usage
  jaguar watch

Example jaguar.yaml
bin:
    'bin/server.dart'

  apis:
    - 'lib/api.dart'

create::interceptor

create::interceptor creates a boilerplate interceptor and its wrapper with given name.

Flags
  • --name: Name of the interceptor

Usage
  jaguar create::interceptor --name SomeInterceptor

create::project

Creates a new project.

Flags
  • --name: Name of the project

Usage
  jaguar create::project --name project_name

create::routegroup

Creates a RouteGroup.

Flags
  • --name: Name of the RouteGroup

Usage
  jaguar create::routegroup --name UserResource
